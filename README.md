# Golang AI Agent - Автономный браузерный агент

AI-агент на Go, который автономно управляет веб-браузером для выполнения сложных многошаговых задач.

## Возможности

- 🤖 **Автономное принятие решений** - агент самостоятельно определяет последовательность действий
- 🌐 **Управление браузером** - полный контроль над веб-браузером через Chromedp
- 🧠 **AI-интеграция** - использует OpenAI API для понимания задач и принятия решений
- 🔄 **Обработка ошибок** - агент адаптируется при неудачных действиях
- 🔒 **Security Layer** - запрашивает подтверждение перед деструктивными действиями
- 🎯 **Под-агенты** - специализированные агенты для разных типов задач
- 💾 **Persistent Sessions** - поддержка сохранения сессий браузера

## Установка

1. Клонируйте репозиторий:
```bash
git clone https://github.com/Angabebr/Golang-AI-agent.git
cd Golang-AI-agent
```

2. Установите зависимости:
```bash
go mod download
```

3. Создайте файл `.env` на основе `.env.example`:
```bash
cp .env.example .env
# Затем отредактируйте .env и добавьте ваш реальный API ключ
```

**⚠️ ВАЖНО:** Никогда не коммитьте файл `.env` в репозиторий! Он уже добавлен в `.gitignore`.

## Использование

Запустите агента:
```bash
go run main.go
```

Агент откроет браузер и будет ждать ваших команд в терминале.

### Примеры задач

1. **Удаление спама:**
   ```
   Прочитай последние 10 писем в яндекс почте и удали спам
   ```

2. **Заказ еды:**
   ```
   Закажи мне BBQ-бургер и картошку фри из того места, откуда я заказывал на прошлой неделе
   ```

3. **Поиск вакансий:**
   ```
   Найди 3 подходящие вакансии AI-инженера на hh.ru и откликнись на них с сопроводительным письмом
   ```

## Архитектура

- `main.go` - точка входа
- `agent/` - основная логика агента
- `browser/` - управление браузером
- `ai/` - интеграция с OpenAI
- `context/` - управление контекстом и токенами
- `security/` - security layer для деструктивных действий
- `subagents/` - специализированные под-агенты

## Требования

- Go 1.21+
- Chrome/Chromium браузер
- OpenAI API ключ

## 🔒 Безопасность

**ВАЖНО:** Проект настроен для защиты чувствительных данных:

- ✅ Файл `.env` автоматически игнорируется git (не попадет в репозиторий)
- ✅ Директория `browser_data/` игнорируется (содержит cookies и сессии)
- ✅ Все API ключи хранятся только в локальном `.env` файле
- ✅ Используйте `.env.example` как шаблон, но **никогда** не коммитьте реальные ключи

**Если вы случайно закоммитили `.env` с реальными ключами:**
1. Немедленно сгенерируйте новый API ключ на OpenAI
2. Удалите старый ключ из истории git: `git filter-branch` или используйте BFG Repo-Cleaner
3. Обновите `.env` с новым ключом
