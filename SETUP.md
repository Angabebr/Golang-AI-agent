# Настройка AI-агента

## Требования

1. **Chrome или Chromium** должен быть установлен
2. **Файл .env** с настройками
3. **Отдельная директория для браузера** (рекомендуется)

## Создание файла .env

Создайте файл `.env` в корне проекта со следующим содержимым:

```env
OPENAI_API_KEY=your_api_key_here
OPENAI_MODEL=gpt-4-turbo-preview
BROWSER_USER_DATA_DIR=./browser_data
ENABLE_SECURITY_LAYER=true
START_URL=https://www.google.com
```

### Важные настройки:

- **OPENAI_API_KEY** - ваш API ключ от OpenAI (обязательно)
- **BROWSER_USER_DATA_DIR** - директория для данных браузера (ВАЖНО: используйте отдельную директорию!)
  - ✅ Хорошо: `./browser_data` или `C:\MyProject\browser_data`
  - ❌ Плохо: `C:\Users\YourName\AppData\Local\Google\Chrome\User Data` (стандартная директория Chrome)

## Почему браузер может закрываться?

### 1. Использование стандартной директории Chrome

Если вы используете стандартную директорию Chrome (`C:\Users\...\AppData\Local\Google\Chrome\User Data`), это может вызвать проблемы:
- Chrome может быть уже запущен
- Конфликты доступа к файлам
- Браузер может закрываться автоматически

**Решение:** Используйте отдельную директорию, например `./browser_data`

### 2. Антивирус или брандмауэр

Антивирус может блокировать запуск Chrome или закрывать его.

**Решение:** Добавьте исключение для `Golang-AI-agent.exe` и директории `browser_data`

### 3. Недостаточно прав

Программа должна иметь права на:
- Запись в директорию `browser_data`
- Запуск Chrome/Chromium

**Решение:** Запустите программу от имени администратора или проверьте права доступа

## Проверка настройки

1. Убедитесь, что файл `.env` существует в корне проекта
2. Проверьте, что `BROWSER_USER_DATA_DIR` указывает на отдельную директорию
3. Убедитесь, что Chrome полностью закрыт перед запуском агента
4. Проверьте права доступа к директории браузера

## Пример правильной настройки

```env
# .env файл
OPENAI_API_KEY=sk-...
OPENAI_MODEL=gpt-4-turbo-preview
BROWSER_USER_DATA_DIR=./browser_data
ENABLE_SECURITY_LAYER=true
START_URL=https://www.google.com
```

Директория `browser_data` будет создана автоматически при первом запуске.

## Если проблема сохраняется

1. Закройте все окна Chrome
2. Удалите директорию `browser_data` (если она существует)
3. Запустите программу снова
4. Проверьте логи в консоли на наличие ошибок
